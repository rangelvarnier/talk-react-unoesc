Performance na web com react

Problema 
* Em 2 anos tivemos um aumento de 105% nos usuários Light Web.
* Média de 1 novo cliente a cada 2 dias.
* Aumento da base de usuários e dados gerados.
* Usuário com poder de operação maior.
* Funcionalidades complexas, com maior exigencia visual. 

Imagens tabela de preço ou mapa de reserva

Causa
* Conjunto de dados maiores.
- Consultas Sql passam a ter custo maior para obter as informaṍes.
- Trafego de rede aumenta.
- Memória e processamento do servidor.
* Visializações que eram simples ficam complexas.
* Carregamento das páginas mais lento.

Efeito
* Insatisfação do usuário final.
* Insatisfação da equipe com o resultado do produto.
* Custos para manter infraestrutura.

Conhecendo a infraestrutura
* Aplicação padrão MVC.
* Model: Acesso a dados.
* View: Templating engine HTML/CSS/JS.
* Controller: Processamentos e fluxo de negócio.

Evolução natural
* Templating engine VS Vanilla Js.
* Views mais exigentes.
* Apresentação dinâmica.

Exemplo com Adonis (buscar dados no endpoint e exibir com js, template string)

Até agora nada de React
* Provocamos mudanças de arquitetura e comportamento da aplicação
* Menos processamento no server
* Mais processamento no client
* Resolvemos problemas de trafego de rede
* Componentização
* Separamos código de lógica do código de exibição
(exemplo do template string e uma função que processo algum tipo de dado chamando a view)

Ainda continuamos com um problema
* Todo conteúdo Js é blocante e sequencial
  * O navegador para a renderização enquanto executa um script
  * Antes da renderização o navegador cria uma árvore DOM analisando HTML.
* Nós DOM custam caro
  * Uma tabela com 100 linhas e 20 colunas geram 2000 nós DOM.

O que é o React e como ele pode nos ajudar?

* Idealizado pela equipe do Facebook e Instagram
* É uma biblioteca para criar interfaces

* Funciona como o V do MVC, permitindo criar nossos próprios componentes
* Permite quebrar diferentes elementos em componentes reutilizáveis.

Aaa é tipo o Angular né?

NÃÃÃO
* Angular é um framework, ou seja um conjunto de ferramentas.
* React é uma lib que resolve um problema específico, renderizar componentes.

E esse tal de virtual DOM?
* Inicializado o componente o metodo render gera uma representação da view e injeta no documento
* Quando algum dado muda, o metodo render é chamado novamente
* O React faz um diff entre o velho e o novo valor e aplica no DOM
